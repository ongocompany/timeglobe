# 🌐 프로젝트 개발 계획서: TimeGlobe (웹 & 모바일 앱)

## 1. 프로젝트 개요

- **목적:** 지질학적 시대부터 인류 역사까지 지구의 변화를 3D 지구본과 타임라인으로 탐험하는 인터랙티브 크로스 플랫폼(Web/App) 서비스.
    
- **타겟 사용자:** 역사/과학 관심층, 학생, 교육 종사자.
    
- **핵심 기술 방향:** AI 코딩 도구를 활용한 1인 개발. **Supabase**를 활용한 동적 데이터 관리 및 향후 모바일 앱 출시를 고려한 확장성 있는 아키텍처 구축.
    

## 2. 유사 서비스 분석 및 차별화 전략

| **서비스명**        | **주요 특징**                  | **한계점**                            |
| --------------- | -------------------------- | ---------------------------------- |
| **Chronas**     | 방대한 위키데이터 기반 2D 역사 지도.     | 정보량이 과도하여 UI가 복잡하고 일반인의 진입 장벽이 높음. |
| **Geacron**     | 연도 검색 기반의 2D 세계 국경선 지도.    | 시각적 디자인이 투박하고 UX가 다소 단조로움.         |
| **EarthViewer** | 3D 기반 지질 시대 판 구조론 및 기후 변화. | 인류 역사(국경, 문명, 사건)와의 결합이 없음.        |

### 💡 TimeGlobe만의 핵심 차별점 (경쟁력)

1. **지질학-역사의 3D 통합:** 생물종의 진화와 대륙 이동부터 현대 문명까지, 하나의 3D 지구본 위에서 끊김 없이 이어지는 독보적인 시각적 경험.
    
2. **정보의 선택과 집중 (큐레이션):** 방대한 데이터를 쏟아내지 않고, 시대별 '핵심 사건' 위주로 직관적인 마커와 모달 창을 제공하여 피로도 최소화.
    
3. **아시아/한국사 딥다이브:** 서구권 중심의 기존 지도들과 달리, 삼국시대의 세밀한 변화나 당나라(當時)의 정확한 영토 범위, 주요 인물(이백, 두보 등)의 족적 등 동양사 데이터를 풍부하게 담아내어 국내 사용자에게 강력한 어필.
    

## 3. 기술 스택 및 개발 환경 (동적 & 앱 확장 고려)

- **프론트엔드 (웹 & 앱 기반):** **React (또는 Next.js)**
    
    - _이유:_ 컴포넌트 관리가 편하고, 향후 모바일 앱(React Native)으로 코드를 이식하거나 하이브리드 앱으로 패키징하기 가장 유리합니다. AI 코딩 도구들도 React 코드를 매우 잘 짜줍니다.
        
- **3D 렌더링:** **Globe.gl** (지구본 구현 및 GeoJSON, 마커 렌더링 최적화)
    
- **백엔드 & DB:** **Supabase** (PostgreSQL)
    
    - 시대별 이벤트, 영토 데이터(GeoJSON), 사용자 결제 정보 등을 동적으로 관리.
        
- **모바일 앱 패키징 전략:** 웹 버전을 먼저 완성한 후, **Capacitor**나 **PWA (Progressive Web App)** 기술을 사용해 웹 코드를 그대로 모바일 앱(iOS/Android)으로 껍데기만 씌워 스토어에 출시 (1인 개발 리소스 최소화).
    

## 4. 수익 창출 모델 (Monetization Strategy) 검토

회원님의 직관대로 **'MVP 제한 노출 + 풀 콘텐츠 일회성 결제 (Freemium)'** 방식이 가장 이상적입니다.

- **광고 모델의 치명적 단점:** 우주를 유영하는 3D 지구본의 몰입감이 핵심인 서비스인데, 배너나 팝업 광고가 들어가면 사용자 경험(Vibe)이 완전히 망가집니다. 트래픽이 엄청나지 않는 한 수익도 미미합니다.
    
- **구독 모델의 한계:** 역사 데이터는 매일 새롭게 업데이트되는 성격이 아니므로, 사용자가 매월 돈을 지불할 동기가 부족합니다.
    
- **👑 권장 모델 (Freemium + One-time Pass):**
    
    - **무료 제공 (MVP):** 서기 0년 ~ 1500년 (예: 로마 제국 ~ 대항해시대) 구간만 무료로 개방하여 흥미 유발 및 바이럴 유도.
        
    - **유료 결제 (Time-Travel Pass):** 1회성 결제(예: 4,900원~9,900원)를 통해 '지질 시대(공룡, 판게아)' 및 '현대사(1, 2차 세계대전 등)'의 잠금 해제. 교육용 프리미엄 퀄리티를 유지하며 확실한 전환 유도.
        

## 5. 단계별 개발 로드맵

- **1단계: 데이터 설계 (Supabase) 및 디자인 (Figma)**
    
    - Supabase 테이블 스키마 설계 (Eras, Events, Borders 등).
        
    - 반응형(PC/Mobile) UI 및 결제 유도(Paywall) 화면 디자인.
        
- **2단계: 웹 MVP 코어 개발 (AI 바이브 코딩)**
    
    - React 환경 설정 및 Globe.gl 연동.
        
    - Supabase에서 데이터를 불러와 타임라인에 따라 마커와 국경선 렌더링.
        
- **3단계: 핵심 콘텐츠 구축**
    
    - 무료 구간의 데이터(위키백과 요약, 좌표 등)를 Supabase에 입력.
        
    - 지질 시대의 대륙 이동 텍스처 맵 적용 테스트.
        
- **4단계: 결제 시스템 연동**
    
    - Stripe 또는 토스페이먼츠 위젯 연동하여 유료 구간 잠금 해제 로직 구현.
        
- **5단계: 웹 배포 및 모바일 앱 출시**
    
    - Vercel로 웹 정식 배포.
        
    - Capacitor를 통해 앱 스토어(iOS/Android) 심사 및 등록.
