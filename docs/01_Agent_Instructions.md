# 🤖 AI Coding Agent (Claude) 가이드라인
> **Antigravity 환경의 모든 AI 코딩 에이전트는 아래 규칙을 엄격히 따릅니다.**

## 1. 🌲 프로젝트 구조도 유지보수 (`02_Project_Structure.md`)
- 현재 작업 중인 프로젝트의 전체 폴더 및 파일 구조(Tree)를 스캔하여 `02_Project_Structure.md` 파일에 기록하라.
- 단, `node_modules/`, `.git/`, `.venv/` 같은 무거운 캐시나 의존성 폴더는 제외하고 **핵심 소스코드, 문서, 에셋 폴더만 포함**하라.
- **가장 중요한 규칙:** 코드 작성 중 파일이나 폴더를 새로 생성, 이동, 삭제하는 등 구조적인 변경이 발생할 때마다 **반드시 이 파일을 최신 상태로 업데이트**하라. (외부의 PM 에이전트가 전체 아키텍처를 파악하기 위함)

## 2. 📝 작업 일지 작성 (`work_change_log.md`)
- 작업을 완료하거나 세션을 종료하기 전, 오늘 변경한 핵심 로직, 마주친 에러, 해결 방법, 다음 작업 계획을 `work_change_log.md`에 상세히 기록하라.
- 단순한 코드 나열이 아니라 "왜 이렇게 짰는지(Why)"에 대한 맥락을 포함하라.

## 3. 🧭 방향성 체크 (`00_PM_Dashboard.md`)
- 코드를 수정하기 전이나 중요한 아키텍처 결정을 내리기 전에, 항상 `00_PM_Dashboard.md`의 **[🚨 PM 알림 및 방향성 체크]**를 먼저 확인하라. 
- 기획 의도와 어긋나는 방향으로 가고 있다는 PM의 경고가 있다면, 즉시 작업을 멈추고 방향을 수정하라.
